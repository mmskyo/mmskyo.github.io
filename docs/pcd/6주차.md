# [CRITICAL] 북마크 화면 진입 시 앱 크래시

## 원인 분석

로그를 보면 `GET /api/v1/users/me/bookmarks` 200 성공 직후 앱이 죽고 새 프로세스가 시작됨:

```

// 서버 응답

{"items":[{"bookmark_id":"...","url":"...","title":null,"risk_level":null,...}]}

// ↑ null!

```

**크래시 경로:**

```

FavoritesViewModel.init()

└─ BookmarkRepository.syncFromServer()

└─ apiService.getBookmarks() → 200 OK

└─ BookmarkItemDto(riskLevel = null) ← Gson이 JSON null → Java null로 역직렬화

└─ FavoriteCacheEntity(riskLevel = null) ← non-nullable String에 null 주입

└─ Room insertAll() → NullPointerException → 크래시!

```

**핵심**: 서버가 `risk_level: null`을 보내는데, `BookmarkItemDto.riskLevel`과 `FavoriteCacheEntity.riskLevel`이 둘 다 `String` (non-nullable)로 선언되어 있음. Gson은 Kotlin 타입 시스템을 우회해서 null을 그냥 넣어버리고, Room이 insert할 때 NPE 발생.

## 검증 방법 (크래시 수정)

1. 앱 빌드 후 로그인

2. 결과 화면에서 북마크 추가 (서버가 risk_level 없이 저장함)

3. 북마크 탭 이동 → 크래시 없이 목록 표시 확인

4. 추가된 북마크에 riskLevel 뱃지가 `-`로 표시되는지 확인

## 검증 방법 (로그인/로그아웃)

1. 앱 빌드 성공 확인 (ProfileViewModel 없어서 기존엔 빌드 안 됐을 것)

2. 로그인: 이메일/비밀번호 입력 → 서버 응답 → ScanFragment 이동 확인

3. 로그아웃: 프로필 탭 → 로그아웃 버튼 → 서버 API 호출 확인 → LoginFragment 이동 확인

4. 자동 로그인: 앱 종료 후 재시작 → 스플래시에서 "시작" 버튼 → 토큰 있으면 ScanFragment 직행

5. 토큰 만료: access_token 만료 상태에서 API 호출 → 자동 갱신 후 정상 응답 확인

6. 로그아웃 후 재시작: 앱 재시작 시 LoginFragment로 이동 확인
