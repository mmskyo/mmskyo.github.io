## 이론

콘솔앱 만들기 소스코드 같은 디렉토리 배치

printf만 추가

``` C++

#include <stdio.h>  
#include <Windows.h>  
  
BYTE b = 0x12;  
WORD w = 0x1234;  
DWORD dw = 0x12345678;  
char str[] = "abcde";  
  
  
int main(int argc, char* argv[]) {  
    BYTE lb = b;  
    WORD lw = w;  
    DWORD ldw = dw;  
    char* lstr = str;  
  
    printf("바이트값 : %d 워드값 : %hu\n", b, w);  
    printf("Hello World!\n");  
  
    return 0;  
}  

```

  

디버거로 메인함수 찾기

메인함수들어가기

우리가 저장한 바이트를 보려면 어떻게해야하나

  

바이트추력으로

메인 걱로가서 주소로 이동

hexa decimal 두자리씩 저장됨

덤프 주소에 하나가 저장(00에 데이터저장)

저장되는 방식 빅 vs 리틀

학교컴퓨터는 리틀 엔디언

사이즈 1 일때 12

2칸 12 34

4칸 12 34 56 78

문자열 문자는 5개지만 6칸 맨뒤 널값이 들어가야해서 \n 

61 62 63 64 65 00

  

리틀 엔디언은 거구로 저장

미리 타입을 선언하지만 공간을 전부 쓰는 것이 아님 워드(2칸)를선언할때 12 만 쓴다면 

빅엔디언은 몇번째칸에 너흘지 고민해야함

리틀은 무조건 넣ㄱ만하면됨 12만넣으면 그냥 write

아랫자리수부터 넣기에 메모리 쓰기가 좋아짐

컴퓨터는 연산을 뒤부터 하니까
